# 📦 Versions & Releases

Documentation sur le versioning et les releases du projet.

## Fichiers

- **VERSIONING.md** - Stratégie de versioning (SemVer)
- **RELEASES.md** - Historique des releases et versions

## Semantic Versioning

Le projet suit **SemVer** format `MAJOR.MINOR.PATCH`:

- **MAJOR** (1.0.0 → 2.0.0): Breaking changes
- **MINOR** (1.0.0 → 1.1.0): Nouvelles features
- **PATCH** (1.0.0 → 1.0.1): Bug fixes

## Release Schedule

- Versions stables tous les mois (1er du mois)
- Release candidates 1 semaine avant
- Hotfixes dès que besoin (patch version)
